# VALIDATION.md

## Validation strategy

Every implementation must define and run applicable validation.

## Minimum checks

- Backend tests.
- Frontend/mobile tests when applicable.
- Static analysis/lint when available.
- Database migration validation.
- Docker Compose startup when relevant.
- Manual demo path for voice order confirmation.
- Realtime event delivery validation.

## Critical acceptance validation

AI-generated orders must require human confirmation before submission.
