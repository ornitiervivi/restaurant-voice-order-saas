# Web Console

Flutter Web application planned for restaurant admin, kitchen and bar screens.

## Current state

This directory is intentionally a skeleton for TASK T-001. No Flutter project has been generated yet.

## Intended responsibilities

- Admin setup for restaurant profile, users, tables and products.
- Open-order monitoring for admins.
- Kitchen and bar station screens for confirmed items only.
- Realtime status updates through authenticated restaurant-scoped channels.

## Stack boundary

- Target stack: Flutter Web for the MVP unless a future decision changes it.
- Kitchen/bar screens must never receive AI-generated drafts before waiter confirmation.
